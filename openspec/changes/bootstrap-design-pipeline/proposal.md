# Proposal: bootstrap design-pipeline

## Why

Design and frontend AI skills are powerful but scattered. Users need a design-first pipeline that can govern optional skills, preserve artifacts, support motion design, and run without a human watching every intermediate UI state.

## What

- Create an open-source repository for `design-pipeline`.
- Port the current Codex skill into `skill/`.
- Align the project with OpenSpec-style specs and changes.
- Define the initial long-lived design-pipeline spec.
- Include self-check, QA, motion, agent handoff, curation, and open-source readiness references.

## Non-Goals

- Do not build a generic agent marketplace.
- Do not require all optional companion skills.
- Do not require GBrain or OpenSpec CLI to be installed for the core pipeline to work.

## Impact

This change establishes the repository foundation and source-of-truth specification.
