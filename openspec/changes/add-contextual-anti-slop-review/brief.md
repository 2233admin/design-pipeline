# Brief: contextual anti-slop review

## Goal

Internalize the useful parts of an external anti-slop design law as a bounded, evidence-backed QA
capability.

## Audience

Designers, developers, and coding agents using design-pipeline for product UI, marketing pages,
website references, and component systems.

## Surface

- Stage 2 design directions
- Stage 3 `design.md` and project `DESIGN.md` decisions
- Stage 6 `qa.md`
- Local evaluator output

## Constraints

- No external runtime dependency.
- No remote prompt execution or global instruction mutation.
- No unlicensed source-text vendoring.
- Existing companion taste skills remain optional and authoritative only within their documented
  capability boundaries.

## Acceptance

- A machine-readable rubric distinguishes hard defects from contextual and preference concerns.
- Missing or unverified hard evidence blocks.
- Contextual exceptions are explicit and evidence-backed.
- Brand-specific black, cyan, red, editorial composition, or hard-edged geometry do not fail only
  because an external taste prompt dislikes them.
- Dual-site fixtures demonstrate the distinction between real verification gaps and stylistic false
  positives.
